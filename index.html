MARLIN

An autonomous surface vehicle for proactive drowning prevention — built under $500. Believed to be the first proprietary ASV at this price point capable of real-time human detection via computer vision on constrained hardware.

The vision system runs a FOMO-based ML model on an ESP32-S3. Navigation uses GPS path recording and playback, heading-based differential thrust via the HMC5883L magnetometer, MPU6050 IMU for stability, and LiDAR for obstacle avoidance. Motor control runs through a PCA9685 with iBus RC passthrough for manual override.

Recognized by SwRI, IEEE, and the U.S. Navy. A sole-author paper on CV-based drowning detection on severely constrained embedded hardware is forthcoming.

TRIPPY

A Sawppy rover variant built at Trinity University to give engineering students a robust open-source hardware platform. TRIPPY ("Trinity Robotics Instructional Planetary Platform") is an adaptation of Roger Chen's SAWPPY design with several hardware modifications — including custom LX-224 servo brackets, reinforced steering knuckles with M3 nut captures, and laser-engraved electronics layout panels.

I joined as the software lead, responsible for building out TRIPPY's autonomous navigation. The stack runs ROS 2 Jazzy on a Raspberry Pi 5 with Ubuntu 24.04. I integrated a Kinect RGB-D sensor via kinect_ros2 for pointcloud and depth data, ported the simulation from Gazebo Classic to Gazebo Harmonic, and am now working up through the Nav2 stack toward full autonomous navigation.

Current focus: implementing a patrolling behavior — a defined route with an "interesting detour" mode that deviates to investigate nearby objects before returning to the patrol path.
